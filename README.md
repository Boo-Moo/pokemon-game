## 🎮 游戏介绍

《精灵捕捉大师》是一款回合制精灵收集游戏。玩家扮演一名训练家，在野外草丛中寻找并捕捉各种精灵。通过攻击降低精灵的血量可以提高捕捉成功率，成功捕捉后可以获得分数，用于购买更多精灵球，继续收集更多的精灵。

### 游戏特色

- **🎨 双显示模式**：支持文字模式和图形模式，可根据喜好自由切换
- **📖 完整图鉴系统**：记录所有已收服的精灵，支持查看详细资料
- **⚔️ 回合制战斗**：经典的攻击-捕捉-逃跑战斗机制
- **🛒 商店系统**：使用捕捉获得的分数购买精灵球
- **📊 统计系统**：查看图鉴完成度、属性分布等统计数据
- **💾 自动保存**：游戏进度自动保存，下次可继续游戏

## 💻 安装说明

### 系统要求
- Python 3.8 或更高版本
- Tkinter（Python标准库，通常已内置）
- 操作系统：Windows / macOS / Linux

### 安装步骤

1. **克隆或下载项目**
```bash
git clone https://github.com/Boo-Moo/pokemon-game.git
cd pokemon-game


2. **创建虚拟环境（推荐）**
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS/Linux
python3 -m venv venv
source venv/bin/activate
```

3. **安装依赖**
```bash
pip install -r requirements.txt
```

4. **运行游戏**
```bash
python main.py
```

### 依赖包说明

项目主要依赖Python标准库，无需额外安装。如需图形增强功能，可安装：

```txt

## 🚀 快速开始

### 启动游戏
1. 运行 `python main.py`
2. 输入训练家名字
3. 选择显示模式（文字模式/图形模式）
4. 点击"开始冒险"

### 基本操作

| 操作 | 说明 |
|------|------|
| 🌿 进入草丛 | 遭遇野生精灵 |
| ⚔️ 攻击 | 降低野生精灵血量 |
| 🎾 扔精灵球 | 尝试捕捉野生精灵 |
| 🏃 逃跑 | 逃离战斗 |
| 🛒 商店 | 购买精灵球（10分/个） |
| 📖 图鉴 | 查看已收服的精灵 |
| 🎨 切换模式 | 切换显示模式 |

### 游戏流程

1. **进入草丛** → 随机遇到野生精灵
2. **选择行动**：
   - 攻击：降低精灵血量，提高捕捉率
   - 捕捉：使用精灵球尝试收服
   - 逃跑：离开战斗
3. **捕捉成功** → 获得10分，精灵加入图鉴
4. **购买精灵球** → 使用分数在商店购买
5. **继续冒险** → 收集更多精灵

## 🎯 游戏玩法

### 捕捉机制

捕捉成功率计算公式：
```
捕捉率 = 基础捕捉率 × (1 - 当前HP/最大HP × 0.5)
```

- 基础捕捉率：每种精灵固定（0.2 ~ 0.7）
- HP越低，捕捉成功率越高
- 最高成功率为85%

### 精灵属性

| 精灵 | 属性 | HP | 攻击 | 捕捉率 |
|------|------|-----|------|--------|
| 皮卡丘 | 电系 | 50 | 15 | 40% |
| 小火龙 | 火系 | 60 | 18 | 35% |
| 杰尼龟 | 水系 | 70 | 12 | 45% |
| 妙蛙种子 | 草系 | 65 | 14 | 40% |
| 波波 | 飞行系 | 40 | 10 | 60% |
| 可达鸭 | 水系 | 55 | 13 | 50% |
| 卡比兽 | 一般系 | 100 | 20 | 20% |
| 伊布 | 一般系 | 45 | 12 | 50% |

### 分数系统

- 捕捉成功：+10分
- 精灵球价格：10分/个
- 初始精灵球：15个

## 📁 项目结构

```
pokemon-game/
│
├── main.py                 # 游戏入口
├── config.py               # 配置文件
├── requirements.txt        # 依赖列表
│
├── models/                 # 数据模型层
│   ├── pokemon.py         # 精灵模型
│   ├── player.py          # 玩家模型
│   └── battle.py          # 战斗模型
│
├── strategies/             # 策略模式（显示策略）
│   ├── display_strategy.py    # 显示策略抽象基类
│   ├── text_strategy.py       # 文字显示策略
│   └── graphic_strategy.py    # 图像显示策略
│
├── views/                  # 视图层（GUI界面）
│   ├── login_view.py      # 登录界面
│   ├── game_view.py       # 主游戏界面
│   └── shop_view.py       # 商店界面
│
├── controllers/            # 控制层
│   └── game_controller.py # 游戏主控制器
│
├── factories/              # 工厂模式
│   └── pokemon_factory.py # 精灵工厂
│
├── data/                   # 数据文件
│   └── pokemon_data.py    # 精灵数据
│
└── utils/                  # 工具类
    └── helpers.py         # 辅助函数
```

### 架构说明

项目采用经典的MVC架构，并结合设计模式：

- **Model（模型层）**：管理数据和业务逻辑
  - `Pokemon`：精灵数据模型
  - `Player`：玩家数据模型
  - `Battle`：战斗逻辑模型

- **View（视图层）**：负责界面显示
  - `LoginView`：登录界面
  - `GameView`：主游戏界面
  - `ShopView`：商店界面

- **Controller（控制层）**：协调Model和View
  - `GameController`：游戏主控制器

- **Strategy Pattern（策略模式）**：显示策略
  - `TextDisplayStrategy`：文字显示
  - `GraphicDisplayStrategy`：图形显示

- **Factory Pattern（工厂模式）**：创建精灵对象
  - `PokemonFactory`：精灵工厂

## 🔧 扩展开发

### 添加新精灵

1. 在 `data/pokemon_data.py` 中添加精灵数据：
```python
POKEMON_DATABASE["新精灵"] = {
    "id": 9,
    "name": "新精灵",
    "hp": 55,
    "attack": 14,
    "catch_rate": 0.45,
    "emoji": "🌟",
    "description": "精灵描述",
    "element": ElementType.NORMAL,
    "image_type": "default",
    "color": "#FF69B4"
}
```

2. 在 `factories/pokemon_factory.py` 的 `_initialize_templates` 方法中注册

### 添加新的显示策略

1. 继承 `DisplayStrategy` 抽象类
2. 实现 `create_wild_display`、`update_wild_display` 等方法
3. 在 `GameController.set_display_mode` 中注册

### 添加新功能

- **进化系统**：在 `models/pokemon.py` 中添加进化方法
- **道具系统**：创建 `models/item.py` 和 `views/item_shop.py`
- **对战系统**：在 `controllers/battle_controller.py` 中扩展

## ❓ 常见问题

### Q: 游戏无法启动怎么办？
A: 检查Python版本是否为3.8以上，确保Tkinter已安装。

### Q: 图形模式不显示图像？
A: 图形模式使用Canvas绘制，不需要额外图像文件。如不显示，请切换到文字模式测试。

### Q: 如何保存游戏进度？
A: 游戏会自动保存到 `player_名字.json` 文件中，下次启动会自动加载。

### Q: 可以添加更多精灵吗？
A: 可以！按照"扩展开发"部分的说明添加即可。

### Q: 游戏卡顿怎么办？
A: 建议切换到文字模式，或关闭其他占用资源的程序。
